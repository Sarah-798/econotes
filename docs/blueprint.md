# **App Name**: EcoNote

## Core Features:

- Note Creation: Users can create and save text notes with an accessibility-first design.
- Speech-to-Text Input: Enable speech-to-text for hands-free note creation using device microphone. The note content is then stored.
- Text-to-Speech Playback: Read notes aloud using text-to-speech, enhancing accessibility for users who prefer auditory learning.
- Sentiment Analysis: Use sentiment analysis as a tool to classify the tone of the notes.
- Map Integration: Associate notes with geographical locations using Mapbox GL JS and Geolocation API. Allow the user to save geolocation to the notes.
- Paper Saved Counter: Display a counter showing the number of 'Paper Saved' based on the digital notes created.
- Cloud Sync: Utilize Firestore database to ensure users' notes sync across their devices and are safely stored.

## Style Guidelines:

- Primary color: Forest green (#386641) to symbolize sustainability and nature.
- Background color: Light beige (#F5F5DC) to provide a paper-like, eco-friendly feel and good contrast.
- Accent color: Earthy brown (#795548) for buttons and interactive elements, grounding the interface.
- Body and headline font: 'PT Sans', a humanist sans-serif for readability and a modern, accessible feel.
- Use eco-friendly icons representing notes, maps, and accessibility features. Consider line icons for simplicity.
- Implement an accessibility-first layout with a focus on simplicity, large fonts, and clear hierarchy. Ensure compatibility with screen readers and keyboard navigation.
- Incorporate subtle animations on interactions such as saving a note or switching themes to provide visual feedback without being distracting.
